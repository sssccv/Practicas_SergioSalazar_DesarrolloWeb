DiagramaRelacional

Modelo Entidad-Relacion

  • Cliente
    • Atributos: ID_Cliente, Nombre, Dirección, Teléfono, Email
  • Producto (Propiedades)
    • Atributos: ID_Producto, Descripción, Precio, Stock, Estado (disponible, vendido, etc.)
  • Carrito de compras (Compra)
    • Atributos: ID_Carrito, Fecha, Total
  • Orden (Compra confirmada)
    • Atributos: ID_Orden, Fecha_Compra, Total, Estado (pagado, pendiente, cancelado)
  • Pago
    • Atributos: ID_Pago, Monto, Fecha_Pago, Método
  • Relaciones
    • Cliente - Carrito de compras: Un cliente puede tener uno o más carritos, pero generalmente uno activo.
    • Carrito de compras - Producto: Un carrito puede contener varios productos, y un producto puede estar en varios carritos (relación de muchos a muchos, con una tabla intermedia "DetalleCarrito").
    • Carrito de compras - Orden: Cuando el cliente realiza la compra, se genera una orden.
    • Orden - Pago: La orden se paga mediante un pago asociado.
